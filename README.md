# MusicFlow - 音乐律动沉浸式体验

一款基于 Three.js 的音乐可视化与歌词渲染工具，支持 AI 智能分析、多种视觉效果和精美歌词特效。

## 功能特性

### 音频播放
- 支持 MP3、WAV、FLAC、OGG、M4A、AAC 等主流音频格式
- 支持从文件名自动解析歌曲名和艺术家信息（格式：`艺术家 - 歌曲名`）
- 完整的播放控制：播放、暂停、停止、进度跳转

### AI 音乐分析
- 集成阿里云通义千问 API（Qwen Turbo/Plus/Max）
- 智能分析音乐风格、氛围、情绪
- 自动推荐背景效果和颜色主题
- 支持识别高潮点，自动触发特效增强
- 分析结果缓存，避免重复调用

### 视觉效果
| 效果 | 说明 |
|------|------|
| **极光** | 渐变色带，带有波动和发光效果 |
| **星空** | 闪烁的星星粒子，随节奏闪烁 |
| **雨滴** | 白色细长雨滴垂直下落 |
| **飘雪** | 雪花缓慢飘落，带轻微摆动 |
| **简约** | 仅保留微弱光斑装饰 |

- 支持开关背景光晕网格
- 可调节效果强度（30%-200%）
- 视觉效果随音频节拍实时响应

### 歌词系统
- 支持 LRC 格式歌词文件
- 多种歌词特效：渐变、发光、粒子、简约
- 多种跟随模式：跳动（卡拉OK）、渐变、无效果
- 6 种字体可选：雅黑、黑体、宋体、仿宋等
- 可调节歌词切换速度

### 颜色主题
- 8 种预设颜色主题
- 支持自定义主题颜色

## 快速开始

### 环境要求
- Node.js 18+
- npm 或 yarn

### 安装依赖

```bash
npm install
```

### 启动开发服务器

```bash
npm run dev
```

访问 http://localhost:5173 查看应用。

### 构建生产版本

```bash
npm run build
```

构建产物在 `dist/` 目录。

## Docker 部署

### 使用 Docker Compose（推荐）

```bash
# 构建并启动
docker compose up -d

# 查看日志
docker compose logs -f

# 停止服务
docker compose down
```

访问 http://localhost:3000

### 手动构建

```bash
# 构建镜像
docker build -t musicflow .

# 运行容器
docker run -d -p 3000:80 --name musicflow musicflow
```

## 使用指南

### 基础使用

1. **导入音频**：点击「选择音频文件」按钮，选择本地音频文件
2. **导入歌词**（可选）：点击「选择歌词文件」按钮，选择 LRC 格式歌词
3. **播放音乐**：点击「播放」按钮开始体验

### AI 智能分析

1. **配置 API Key**：在设置中填入阿里云通义千问 API Key
2. **选择模型**：可选择 Turbo（快速）、Plus（标准）或 Max（最强）
3. **分析音乐**：点击「AI 分析音乐」按钮，等待分析完成
4. **自动应用**：分析完成后自动应用推荐的颜色主题和背景效果

### 视觉设置

| 设置项 | 说明 |
|--------|------|
| 背景效果 | 选择极光/星空/雨滴/飘雪/简约 |
| 背景光晕 | 开关背景网格光晕效果 |
| 颜色主题 | 选择预设颜色或自定义 |
| 效果强度 | 调节粒子数量和效果强度 |

### 歌词设置

| 设置项 | 说明 |
|--------|------|
| 歌词特效 | 选择渐变/发光/粒子/简约效果 |
| 跟随模式 | 选择跳动/渐变/无效果 |
| 字体 | 选择雅黑/黑体/宋体/仿宋 |
| 切换速度 | 调节歌词切换动画速度 |

## 技术架构

### 技术栈

| 层级 | 技术 |
|------|------|
| 前端框架 | 原生 JavaScript (ES6+) |
| 3D 渲染 | Three.js |
| 构建工具 | Vite |
| AI 服务 | 阿里云通义千问 API |
| Web 服务器 | Nginx |
| 容器化 | Docker |

### 核心模块

| 模块 | 职责 |
|------|------|
| `scene.js` | Three.js 场景管理，粒子效果，背景渲染 |
| `lyrics.js` | 歌词解析、渲染、特效控制 |
| `analyzer.js` | LLM API 调用，音乐分析 |
| `main.js` | 应用入口，事件绑定，状态管理 |

### 音频分析

使用 Web Audio API 的 `AnalyserNode` 获取音频频率数据：
- 实时获取频率数据用于视觉效果响应
- 计算平均值作为整体节奏强度

## 常见问题

### Q: AI 分析失败怎么办？

1. 检查 API Key 是否正确
2. 确认网络连接正常
3. 尝试切换更快的模型（Turbo）

### Q: 歌词不同步怎么办？

1. 确认歌词文件是标准 LRC 格式
2. 部分歌词文件可能时间戳不准确

### Q: Docker 部署后无法访问？

1. 检查端口是否被占用：`docker compose ps`
2. 查看容器日志：`docker compose logs`
3. 确认防火墙允许对应端口

### Q: 视觉效果不流畅？

1. 降低效果强度
2. 使用更简单的背景效果（简约模式）
3. 关闭其他占用 GPU 的程序
